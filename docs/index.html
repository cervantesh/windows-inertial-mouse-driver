Abstract

This project implements a Windows HID mouse upper-filter driver that treats incoming relative mouse movement as acceleration instead of direct cursor displacement.

The driver keeps virtual velocity in kernel state, emits continued relative movement through a KMDF timer, and applies friction until the cursor settles. The surrounding repository includes the driver source, registry-backed tuning defaults, build scripts, and a Windows setup wizard for local test installation.

03

Implemented Scope

  1. Attach as a HID mouse upper-filter through an INF targeted at the selected mouse hardware ID.
  2. Convert raw relative mouse deltas into persistent X/Y virtual velocity.
  3. Emit continued motion from a KMDF timer while friction decays the stored velocity.
  4. Keep button events tied to physical input so clicks are not repeated during inertial motion.
  5. Package a single-file installer with runtime detection and a traditional Windows wizard flow.

Packaging Evidence

The installer is produced as a small native bootstrapper that embeds the managed setup wizard and driver payload. It checks for the .NET 10 Desktop Runtime, offers assisted installation when missing, and keeps PowerShell windows hidden during driver operations.
